## 3. What Worked ✓

### 3.1 Parser Position Extraction
- **Issue:** Initial benchmarks showed 0% routing - components had no positions
- **Root Cause:** Tests checked wrong attribute (`center` instead of `initial_position`)
- **Fix:** Verified parser correctly extracts positions to `Component.initial_position`
- **Added:** `extract_footprint_positions()` helper function for raw content parsing
- **Tests:** 5 passing tests in `tests/io/test_footprint_position_extraction.py`

### 3.2 Zone Net Detection
- **Issue:** GND (60 pins) failed routing - needs copper pour, not traces
- **Discovery:** Piantor's original PCB uses GND zone on F&B.Cu
- **Fix:** Detect zone nets from `board.zones` and exclude from trace routing
- **Result:** GND correctly excluded, counted as "connected via pour"

```python
# Zone detection code
zone_nets = set()
for z in result.board.zones:
    for net_name in z.net_classes:
        zone_nets.add(net_name)
# → Detected: {'GND'}
```

### 3.3 Pipeline Stage Integration
- **Added:** `LayerAssignmentStage()` for proper layer selection
- **Added:** `NetOrderingStage()` to populate `state.net_order`
- **Result:** Router now finds paths for most keyboard matrix nets

## 4. What Didn't Work ✗

### 4.1 Fine Grid Resolution (0.1mm)
- **Hypothesis:** Finer grid would allow routing through tight spaces
- **Result:** WORSE performance - 12 failed nets vs 4 at 0.5mm
- **Cause:** 25x larger search space causes A* iteration timeout

```
WARNING: Multi-layer A* for /k00 exceeded 109800 iterations (dist=1323 cells)
WARNING: Multi-layer A* for /k20 exceeded 106725 iterations (dist=1325 cells)
```

- **Conclusion:** Grid resolution is NOT the bottleneck

### 4.2 Persistent Failing Nets
Same 4 nets fail at both 0.5mm and 0.25mm:

| Net | Type | Pins | Reference Layer | Our Issue |
|-----|------|------|-----------------|-----------|
| rx | Serial | 2 | F.Cu (3 segments) | Blocked by congestion |
| tx | Serial | 2 | F.Cu (4 segments) | Blocked by congestion |
| VCC | Power | 3 | F.Cu (4 segments) | Should be zone net? |
| /k00 | Keymatrix | 4 | B.Cu (16 segments) | Route order issue |

---

## 5. Root Cause Analysis

### 5.1 Reference Board Comparison
The reference Piantor board successfully routes all nets. Key differences:

1. **Route Order:** Reference may route /k00 FIRST on empty B.Cu
   - Our router routes it LAST after F.Cu is congested
   
2. **Existing Context:** Reference has 237 pre-existing traces
   - We start from scratch with no spatial awareness
   
3. **GND Zone:** Reference uses copper pour on both layers
   - Provides "free" connectivity for 60 pins

### 5.2 A* Iteration Limits
Bidirectional A* has 10,000 iteration limit per segment.
At fine grids (0.1mm), long nets exceed this:
- /k00: 1323 cell distance → 109k iterations (11x over limit)

## 11. Experiment B.2: Fix Retry Iteration Limit

### Objective
Resolve the persistent **109,687 iteration timeout limit** to allow `/k00` to search deeply.

### Hypotheses
1. `MultiLayerAStar` adaptive budget is calculating a low limit due to low base parameters.
2. The limit is being capped by `max_iterations_cap`.

### Investigation & Solution
- **Discovery:** Found that `MultiLayerAStar` (in `multilayer_astar.py`) correctly calculates a 1.75 BILLION iteration budget, but `python_astar.py` (used as a fallback for missing Cython implementation) **ignored** this budget and recalculated its own using default parameters (base=100), resulting in the 109k cap.
- **Fix 1:** Modified `python_astar.py` to respect explicit `max_iterations` from config if provided, disabling its internal adaptive budget.
- **Fix 2:** Modified `python_astar.py`'s internal adaptive estimator to return `self.max_iterations` if it is explicitly high (>200k).
- **Fix 3:** Increased `multilayer_astar.py` `max_iterations_cap` from 200k to 1M.

### Result
- **Budget Enforced:** The router successfully ran with a 1,000,000 iteration limit.
- **Outcome:** `/k00` exhausted the full 1,000,000 iterations and **Failed with "No Path Found"**.
- **Conclusion:** `/k00` is definitively **Physically Blocked**. The failure was not a premature timeout; the router exhaustively searched the reachable space and confirmed no path exists.
